/ Protocol /001 Architecture /

Built different,
by design

Everything emerges from a core set of primitive operations — no orchestration framework required.

Core

Pheromone Signaling

Agents deposit and read stigmergic signals on the hypergraph, creating emergent coordination without central orchestration.

Core

Hypergraph Memory

Knowledge encoded in typed hyperedges allows n-ary relationships that flat graphs cannot represent. Context persists across agent lifecycles.

Protocol

Council Consensus

Disagreements resolved through structured multi-agent deliberation. Minority positions are preserved, not discarded, for future evaluation.

Runtime

Dynamic Skill Trees

Agents discover and load skills at runtime from the shared hypergraph. No restart required. Capability spreads laterally across the swarm.

Network

Federated Clusters

Multiple HSM-II nodes federate via the L5 protocol. Knowledge and agents migrate between clusters with cryptographic identity guarantees.

Ops

Full Observability

Every agent decision, pheromone trail, and council vote is traced. OpenTelemetry export, Prometheus metrics, structured JSON logs.

Five-layer
protocol stack

Separation of concerns from individual cognition through federated network coordination.

L1
Cognitive Layer

Individual agent reasoning, LLM inference, tool dispatch, and local memory management. Each agent is a stateful process with a private working memory.

LLM Runtime Tool Use
L2
Stigmergic Layer

Pheromone deposition and evaporation on the shared hypergraph. Encodes collective intent without explicit messaging between agents.

Pheromones Signals
L3
Hypergraph Layer

Persistent shared knowledge base using typed n-ary hyperedges. Versioned, addressable, and queryable by all agents in the swarm.

Knowledge Memory
L4
Council Layer

Structured deliberation protocol for multi-agent consensus. Handles conflict resolution, skill arbitration, and collective decision-making.

Consensus Voting
L5
Federation Layer

Cross-cluster synchronization, agent migration, and knowledge exchange between independent HSM-II instances over the open federation protocol.

From prompt to
collective answer

1
Spawn

Define the task and spawn your swarm

Declare the goal, the agent roles, and the LLM backends. HSM-II spawns agents and initializes a shared hypergraph namespace for this task.

2
Converge

Agents explore, signal, and cross-pollinate

Each agent reasons independently, deposits pheromone signals on the hypergraph, and reads signals from peers. Emergent coordination follows gradient descent over stigmergic fields.

3
Deliver

Council convenes and delivers a verified result

When convergence is detected, the Council layer convenes to resolve any remaining disagreements. The final answer, with full provenance, is written back to the hypergraph.

Why coordination
must be emergent

Centralized orchestration is an architectural antipattern for LLM agents. It creates a single point of brittleness and prevents the system from developing genuine adaptability. Stigmergic coordination is how complex systems actually work at scale.

AB
Arnaud Bellemare Lead Architect, HSM-II

The key insight is that the hypergraph is not just storage — it is the medium of coordination. Agents don't talk to each other. They talk to the graph, and the graph talks back through pheromone gradients.

HS
Protocol Design L2 / L3 Specification

Federation is not optional. Any serious multi-agent deployment needs to span organizational and geographic boundaries without centralizing trust. L5 gives you that without sacrificing consistency at L3.
